# Security Policy

## Reporting a Vulnerability

If you discover a security issue related to this repository, please avoid opening a public issue with exploit details.

Instead, report it privately to the maintainer with:

- a description of the issue
- steps to reproduce
- impact assessment
- any suggested mitigation

Until a dedicated security contact is published, please use GitHub private maintainer contact where available.

## Scope

This repository is an open-source valuation and agent-integration project. Security-sensitive areas may include:

- credential handling in CI or deployment workflows
- future API integrations
- supply-chain risks in dependencies
- generated artifacts or automation that interact with external systems
